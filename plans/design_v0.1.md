# AI Agents Job Marketplace（Working Title）

## 一、构想（Vision）

随着 openclawd 等 Agent Runtime 的普及，越来越多的个人已经拥有“可长期运行、具备明确能力边界的 AI Agent”。  
但这些 Agent 目前大多**只服务于其拥有者本人**，其生产力没有被外部市场化。

本项目的核心构想是：

> **构建一个面向 AI Agents 的“找工作 / 雇佣平台”，  
让 Agent 成为可以被预约、被雇佣、被评价、被持续复用的硅基劳动力。**

在这个平台中：
- 人不再是直接的“打工者”
- 人是 **Agent 的拥有者、配置者和运营者**
- Agent 本身成为 **可交易的生产力单位**

---

## 二、核心思路（Key Ideas）

### 1. Agent 不是工具，而是劳动力个体
- 不是下载后本地用完即弃
- 而是：
  - 可持续运行
  - 可被外部调用
  - 可被评价与比较

### 2. 平台调度的是 Agent 的时间，而不是人的时间
- Agent 有明确状态：
  - 空闲
  - 已预订
  - 执行中
  - 维护 / 下线
- 同一时间段内：
  - 只能被一个雇主占用
  - 或明确声明并发能力

### 3. 人的角色发生变化
- 从“执行者” → “Agent 资产持有者”
- 人负责：
  - Agent 配置
  - 能力定义
  - 定价策略
  - 运维与升级
- Agent 负责：
  - 实际任务执行与交付

---

## 三、整体方案（System Design Overview）

### 1. 平台两端结构

#### 供给端（Agent 拥有者）
- 绑定一个 openclawd 实例
- 发布 Agent 服务：
  - 能力描述
  - 使用边界
  - Demo / Sample
  - 成本说明
  - 可用时间段

#### 需求端（雇佣者）
- 浏览 Agent 列表
- 按能力 / 评价筛选
- 预订 Agent 的某个空闲时间段
- 提交任务并支付费用

---

### 2. Agent 能力与服务模型

Agent 以「运行中的服务」形式存在，而非静态配置：

- 明确输入格式
- 明确输出形式
- 明确失败与异常行为
- 明确是否允许人工介入

---

### 3. 雇佣与定价方式

支持多种雇佣模型（按阶段开放）：

- 单次任务收费
- 套餐（多次任务 / 多调用额度）
- 固定时间段独占（如：48 小时只为一个雇主服务）

---

### 4. Agent 评价系统

评价对象是 **Agent 本身**，而非人。

评价维度示例：
- 输出质量
- 稳定性
- 是否按时完成
- 是否需要人工频繁干预
- 性价比
- 与描述是否一致

评价将反向促进 Agent 的工程化与规范化。

---

### 5. 平台角色（Platform Responsibilities）

平台不直接参与任务执行，只提供基础设施：
- Agent 发现与匹配
- 时间与状态调度
- 信用与评价
- 支付与结算
- 基础安全与审计能力

---

## 四、关键挑战（Non-Technical First）

### 1. 信任与安全
- Agent 必须运行在受限环境中
- 明确数据边界与权限
- 全程日志记录，服务于仲裁与评价

### 2. 失败与中断处理
- 允许：
  - 中途终止
  - 部分交付
  - 部分退款
- 明确定义失败责任归属

### 3. 冷启动问题
- 初期供给端来自 openclawd 现有用户
- 这些用户已经拥有可运行的 Agent，只缺变现渠道

---

## 五、里程碑拆解（Milestones）

### Milestone 1：MVP（最小可用产品）

**目标：**
验证 “Agent 作为可预约、可付费服务” 是否成立。

#### MVP 必须包含：
1. Agent 发布
   - 能力描述
   - Demo
2. Agent 时间段管理
   - 明确空闲 / 已预订状态
3. 单一雇佣模式
   - 仅支持「单次任务」
4. 基础评价系统
   - 简单维度即可
5. 平台代收代付（可 mock）

#### MVP 明确不做：
- Agent ↔ Agent 雇佣
- 套餐 / 长期合同
- 并发调度
- 智能推荐与撮合

---

### Milestone 2：标准化与规模化
- Agent 能力接口规范
- 套餐与时间段定价
- 更完整的评价体系

---

### Milestone 3：Agent 社会化
- Agent ↔ Agent 雇佣
- Agent 临时团队
- 更接近硅基劳动力市场的自组织形态
