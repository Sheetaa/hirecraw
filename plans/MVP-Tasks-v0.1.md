# MVP-Tasks-v0.1

基于 `plans/MVP-产品交互设计-v0.1.md` 与 `plans/MVP-技术方案设计-v0.1.md` 拆分任务，并按功能依赖安排开发顺序。多个可并行的任务尽量并行。

## 任务拆分

### A. 基础模型与约束
- [x] A1. 定义 Agent/Slot/Booking/Task/Evaluation 数据模型与校验规则
- [x] A2. 定义状态流转（Slot/Booking/Task/Evaluation）与错误码规范

### B. 后端接口（MVP）
- [x] B1. GET `/agents` 列表接口
- [x] B2. GET `/agents/:id` 详情接口（含 slots）
- [x] B3. POST `/bookings` 预订接口（锁 slot + paid 校验）
- [x] B4. POST `/tasks` 任务提交接口（防重复提交）
- [x] B5. GET `/tasks/:id` 任务状态/结果接口
- [x] B6. POST `/evaluations` 评价接口（需任务 done）

### C. 执行与支付模拟
- [x] C1. Mock 执行器（延时 -> 写 output -> 标记 done）
- [x] C2. Mock 支付校验（paid=true）

### D. 前端页面与交互
- [x] D1. `/agents` 列表页（含空/加载状态）
- [x] D2. `/agents/:id` 详情页（含 slots 列表）
- [x] D3. 预订确认弹窗（选择 slot -> 调用 booking）
- [x] D4. `/tasks/:id` 任务提交页（提交后状态展示）
- [x] D5. 任务结果展示（done 输出）
- [x] D6. `/tasks/:id/review` 评价页

### E. 错误处理与交互细节
- [x] E1. 前端错误提示与处理动作（409/422/404/5xx/超时）
- [x] E2. 前端按钮 loading/disabled 与重复提交保护

### F. 文档与验收
- [x] F1. MVP walkthrough（用户流程说明）
- [x] F2. MVP DoD 与已知限制清单

## 完成标准（DoD）

- 至少 1 个 Agent 可展示
- 至少 1 个 Slot 可预订
- 至少 1 个任务可提交
- 至少 1 个结果可返回
- 至少 1 次评价可记录
